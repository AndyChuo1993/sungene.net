Real reason <html lang> stays "en" across all langs (R21 deferred):
  - app/layout.tsx uses headers().get('x-lang')
  - Pages are force-static + generateStaticParams (R18)
  - Build-time render has no request context → headers() empty → fallback en
  - Middleware/proxy DOES set x-lang at REQUEST time, but the prerendered HTML
    served from disk doesn't re-render
  - R19 JS injection in [lang]/layout.tsx sets documentElement.lang client-side
    → Googlebot full-render + screen readers see correct lang
  - Non-JS crawlers (Yandex, Baidu, social bots, IndexNow) see "en"
  - Full structural fix = route-group migration to put <html> under [lang]
    (HIGH RISK, dedicated session, not autonomous-loop safe)
