#3 next.config.mjs images.qualities = [40, 50, 60, 75, 90]
   Next.js 14+ requires the `qualities` allowlist for security — any q value
   not in the list returns 400. Default-only allows q=75. Adding 40/50/60/90
   gives components opt-in path to lower (or higher) quality for specific
   hero / thumbnail / iconographic images.

   Verified post-deploy on /photo-real/library/hero/resources.jpg w=1200:
     AVIF q=50: 11.7 KB  (3× smaller than WebP q=75)
     AVIF q=60: 16.0 KB
     AVIF q=75: 28.7 KB  (already smaller than WebP)
     WebP q=75: 36.3 KB
     q=999:     HTTP 400 (allowlist correctly enforced)

   Note: auditor measured AVIF=218KB for some hero image. Different photo
   from what I tested (resources.jpg → 28KB). Likely the specific photo
   they tested has noise/grain AVIF q=75 handles poorly. With qualities now
   allowing q=50, components can opt into smaller AVIF for those photos
   without changing the default for the rest of the site.
